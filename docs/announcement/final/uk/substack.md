# Мені набридло копіювати контекст між AI coding agents, тому я зробив chorus

Деякий час мій AI coding workflow виглядав продуктивно зовні й трохи абсурдно зсередини.

Я починав у Claude Code, бо він уже був відкритий у репозиторії. Він робив корисну зміну. Потім мені хотілося другої думки, і я відкривав інший термінал, щоб попросити Gemini CLI зробити ревʼю того самого diff. Потім питав Codex про тести або OpenCode про форму рефакторингу.

Результат часто був хороший.

Процес — ні.

Забагато копіювання. Забагато відновлення контексту. Забагато дрібних моментів у стилі "стоп, а який агент бачив які файли?". Це не якась драматична проблема. Просто workflow tax, який накопичується, поки не стає достатньо дратівливим, щоб його прибрати.

Так зʼявився `chorus`.

---

`chorus` — це open-source колекція плагінів для шести AI coding CLI: Claude Code, OpenCode, Gemini CLI, Codex, Cursor та Kilo.

Мета не в тому, щоб замінити будь-який із них. Мені подобається, що ці інструменти мають різні інтерфейси й різні tradeoffs. Мета — зʼєднати їх.

Одна інсталяція. Шість AI agents із вашого улюбленого інструмента.

Ментальна модель — **mesh 6×6**: кожен агент може делегувати задачі будь-якому іншому.

Якщо ви в Claude Code, можна запускати:

```text
/opencode:run Try a simpler implementation of this change.
/gemini:review Review the diff for missed edge cases.
/codex:run Add focused regression tests for the new behavior.
/cursor:run Check if this fits the existing patterns.
/kilo:run Review the naming and long-term readability.
```

Якщо ви в OpenCode, є MCP tools:

```text
delegate_claude
delegate_gemini
delegate_codex
delegate_cursor
delegate_kilo
```

Gemini CLI, Codex, Cursor та Kilo підтримуються через skills і rules.

---

Найкорисніший workflow поки — multi-agent review.

Я не хочу, щоб один агент сказав мені, що мій код хороший. Я хочу, щоб кілька різних агентів атакували зміну з різних боків. Один — на correctness. Інший — на тести. Третій — на зайву складність. Ще один — на відповідність паттернам кодової бази. Ще один — на naming та довгострокову читаємість. Потім я читаю результат і вирішую, що з цього реальне.

Цей останній крок важливий.

Я не намагаюся автоматизувати рішення. Я намагаюся зібрати більше корисної критики перед merge.

AI coding agents дуже добре звучать впевнено. Вони також дуже добре пропускають речі. Але пропускають вони різне. Це і є практична перевага.

Іноді Gemini ловить дивний edge case. Іноді Codex дає кращий план тестів. Іноді Claude пояснює, чому рефакторинг занадто хитрий. Іноді Cursor помічає, що паттерн не вписується в кодову базу. Іноді Kilo вказує на назву, яка заплутає наступного розробника.

Цінність не в тому, що якийсь агент "найкращий". Цінність у тому, що переключатися між ними має бути дешево.

---

У chorus тепер є named workflow команди як окремі плагіни. Multi-agent review, описане вище — `/gemini:review`, `/codex:run`, `/opencode:run` — нікуди не ділося, але тепер є коротший шлях:

```text
/chorus:review        — паралельне review git diff HEAD, одна команда, 5 незалежних думок
/chorus:council       — одне завдання пʼятьом агентам із різними ролями; host синтезує
/chorus:debug         — ранжовані гіпотези першопричини від 5 агентів
/chorus:second-opinion — швидка перевірка одним обраним агентом
```

Пʼять агентів паралельно (Claude, Gemini, Codex, Cursor, Kilo). OpenCode підключений до mesh, але виключений із цих паттернів — TUI stdout не захопити програматично.

OpenCode отримує їх як MCP tools. Gemini CLI, Codex, Cursor та Kilo — як skills і rules. Ідея та сама: не треба щоразу згадувати, які пʼять delegate команд збирати для паралельного review. Одна команда — той самий результат.

Ось філософія дизайну `chorus`: ніякого нового великого дашборда, ніякої нової workflow-релігії, ніякого примусового orchestration layer. Просто команди й інструменти, які дозволяють AI coding CLI спілкуватися між собою.

Я зробив це як практикуючий розробник. Мій use case простий: залишатися в терміналі, тримати свій поточний інструмент і все одно отримувати корисну роботу від інших агентів, коли це потрібно.

Проєкт open source:
**https://github.com/valpere/chorus**

Якщо ви вже використовуєте Claude Code, OpenCode, Gemini CLI, Codex, Cursor або Kilo — `chorus` може влитися у ваш workflow, не вимагаючи його змінювати.

Власне, у цьому й уся суть.
